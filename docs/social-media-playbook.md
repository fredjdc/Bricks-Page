# Bricks Apps: Social Media Playbook

**Objective:** Publish 2 posts per week across Instagram first, then repurpose across LinkedIn and X when useful, without violating the [Brand Voice Guide](brand-voice-guide.md).

Remember: We do not sell with pressure. We state facts. We earn trust through restraint.

The goal is still conversion. The difference is how we get there: sound useful, specific, and human. Some posts can be direct product notes. Some can be small observations. Not every post needs to feel like a campaign.

---

## 1. Weekly Cadence

### Recommended schedule

Post **2 times per week**.

| Day | Time | Primary use |
|---|---:|---|
| **Tuesday** | **1:00 PM** | Feature/value post |
| **Wednesday** | **6:00 PM** | Use case, trust, or product-depth post |

Use the audience's local time zone:

- **US App Store audience:** start with Eastern Time.
- **Peru / LATAM audience:** use Peru time.

### Cadence rule

Do not post randomly. Each post should have one job:

| Goal | Best content type | Example |
|---|---|---|
| Explain value | Carousel | “Know what you’ll pay each month” |
| Build trust | Static / carousel | “No account. Works offline.” |
| Drive install | Product carousel + CTA | “Download Bricks Scan on the App Store.” |
| Educate the market | Single insight post | “A lower payment is not always a cheaper loan.” |
| Show product depth | Feature walkthrough | Refinance, prepayments, search, Auto-Sort |

## 4. Post Shapes

Use these shapes instead of forcing every post into the same structure.

### Shape A: Work Moment

1. **The Moment:** What real work situation is this for?
2. **The Action:** What does the app do?
3. **The Constraint:** Why is the Bricks way better?
4. **The CTA:** A plain next step.

Example:

> The client asks what the payment looks like at a higher rate.  
> Compare both scenarios side by side in Bricks Calc.  
> No account required. Calculations stay on device.  
> Try Bricks Calc on the App Store.

Use this for LinkedIn and conversion-focused posts.

### Shape B: Human Observation

1. Make a short observation about the work.
2. Add one product-specific sentence if useful.
3. Stop.

Example:

> The monthly payment is rarely the whole story. Taxes, insurance, PMI, and rental income change the conversation.

Use this for X and Instagram.

### Shape C: Tiny Lesson

1. Name a practical idea.
2. Give one useful detail.
3. Mention the app only if it fits naturally.

Example:

> A refinance only helps after the break-even point. Bricks Calc compares the current loan with the new option so that point is easier to see.

Use this when the post should build trust before asking for a click.

### Shape D: Product Note

1. Say what changed, what shipped, or what the app does.
2. Keep it casual and concrete.
3. Add a link only when useful.

Example:

> Bricks Calc keeps payment, prepayment, refinance, and rental-income scenarios in one place.

For X, default to Shape B or C. For LinkedIn, default to Shape A or C. For Instagram, default to Shape B with a strong screenshot.

## 10. Missing Content Types to Add

The current screenshot posts are strong product/value posts. Add trust and educational posts so the feed does not feel like repeated product advertising.

### Bricks Calc educational posts

- A lower monthly payment can still cost more.
- Prepayments reduce interest because they reduce principal earlier.
- Refinancing only helps if you pass the break-even point.
- Compare total cost, not just monthly payment.

### Bricks Scan educational posts

- Scanning is not the hard part. Finding the file later is.
- A searchable PDF is more useful than a photo.
- No account required matters when documents are private.
- Auto-organization saves time after the scan.

## 11. Visual Assets Rulebook

Because the copy is minimal, the visuals must carry the weight.

1. **No stock photos:** Do not use photos of people shaking hands, generic buildings, or holding keys.
2. **Use real screenshots:** Show the actual UI. If the copy says "compare loans," the image must show the loan comparison screen.
3. **Keep it clean:** Use a simple, solid-color background using the app’s accent color behind a phone mockup if needed.
4. **Match text and screen:** If the headline says “refinancing,” the image must show the refinance view.
5. **Use one idea per slide:** Do not make one carousel explain every feature.
6. **Keep hierarchy sharp:** Large headline, clear app screen, minimal supporting text.

## 13. Weekly Metrics Review

Do not judge posts by likes first.

Track these weekly:

| Metric | Why it matters |
|---|---|
| Saves | Indicates practical value |
| Shares | Indicates useful or relatable content |
| Profile visits | Indicates intent |
| Link taps | Indicates conversion interest |
| Carousel completion | Indicates message clarity |
| Follows from post | Indicates brand interest |

For Bricks Calc and Bricks Scan, **saves and link taps** matter more than likes.

Review after 4 weeks. Do not change the whole system after one weak post.

## 14. Pre-Posting Checklist

Before publishing, verify:

- [ ] The post has one clear job.
- [ ] The screenshot matches the caption exactly.
- [ ] The CTA is calm and specific.
- [ ] The caption avoids hype words.
- [ ] The hashtags are limited to 3.
- [ ] The post has a matching story version.
- [ ] The post fits the 4-week sequence.
- [ ] The profile bio and pinned posts give enough context.
- [ ] The image does not look like a generic ad.
- [ ] The post is useful even if the viewer does not install the app.

---

## 15. Self-Correction Checklist

- [ ] Did I use the word “seamless,” “ultimate,” or “powerful”? If yes, delete.
- [ ] Is the post about more than one app? If yes, split it into two posts.
- [ ] Does the post name a real work moment or concrete task?
- [ ] Does the CTA sound calm, not pressured?
- [ ] Does the post sound like a reusable template? If yes, make it shorter, more specific, or more observational.
- [ ] Does the image match the text exactly?
- [ ] Could the first sentence stand alone as a useful idea?
